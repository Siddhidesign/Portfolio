IRS2Go App Redesign – UX Case Study

🔍 Phase 1: Discover & Define

▶

🚩 Problem Identification

IRS2Go app provides essential tax functions: refund status, payment, and free tax help.

Despite its value, reviews highlighted:

  • Outdated UX
  • Confusing navigation
  • Web redirections for critical flows (like payments)
  • No task memory (manual re-entry each time)

🔎 Research & Analysis

Methods: App reviews (App Store, Quora), IRS blog insights, competitive benchmarking

Heuristic Evaluation: Identified UI inconsistencies, error-prone forms, and inaccessible input patterns.

Task Analysis: Documented the entire journey for key tasks:

  • Refund status check
  • Payment process
  • Accessing tax help

💬 User Feedback

“It’s frustrating that there’s no guidance if my info doesn’t match IRS records.”
“Entering my details every time is annoying. Where’s auto-fill?”
“The help section is too text-heavy and hard to navigate.”

🎯 Insights

  • Poor efficiency of use
  • Violations of error prevention and recovery
  • No support for recognition over recall

✍️ Phase 2: Ideate & Prototype

▶

🧠 Ideation

Generated wireframes for new task flows with:

  • Fewer steps
  • Simplified input fields
  • Guided instructions & examples

Key focus: Mobile-first clarity, visual hierarchy, reduced user confusion

📱 Low to Mid-Fidelity Prototypes

  • Refund status with pre-filled hints + masked SSN
  • Payment with processor comparison + confirmation modal
  • Help section with FAQ-style structure + voice search

✨ Design Highlights

  • Added sample form guidance for refund inputs
  • Improved accessibility options (text size, color contrast)
  • Integrated user onboarding for first-time users

🧪 Final Phase: Test & Refine

▶

👥 Usability Testing

Participants: 11 users (age 16–65), mix of tech-savvy and first-timers

Tasks: Check refund status, make payment, access tax help

Methods:

  • Think-Aloud Protocol
  • System Usability Scale (SUS)
  • Observation Sheets

📊 Findings

AreaBefore TestingAfter Refinement
LoginConfusing guest/login optionsGuest Access now prominent
PaymentNo preview before paymentAdded confirmation screen
HelpOverwhelming scroll listSearchable FAQs + voice input
RefundPoor instructionsSample form + inline guidance

📈 SUS Score Highlights

  • Overall Usability: ↑
  • Reduced hesitation/confusion
  • Positive response to dropdown improvements, feedback modals, and visual consistency

✅ Final Prototype

Developed in Figma. Includes updated screens for all 3 core tasks, onboarding flow, and refined visuals based on user feedback.

🎉 Key Takeaways

  • Even a government app like IRS2Go benefits hugely from UX-focused redesign.
  • Simple changes (auto-fill, confirmation, voice search) massively improve trust and ease.
  • Systematic user testing revealed deep gaps not visible in initial audits.
