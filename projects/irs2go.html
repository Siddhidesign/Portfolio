How It Started

IRS2Go helps users check refunds, make payments, and find tax help — but suffers from poor UX.

🚩 Problem Identification

  • Outdated UX
  • Confusing navigation
  • Web redirections break flow
  • No task memory (SSN, refund amount)

💬 User Quotes

This refund tool is useless. I put everything in and it says no match!
App Store User
Why do I have to re-enter all my info every single time?
User Interview

Frustration and confusion were common in user feedback.

Research & Scope

🔎 What We Did

  • Reviewed app stores, Quora, blogs
  • Ran heuristic evaluation
  • Mapped 3 key tasks: refund, pay, help

🎯 Key Insights

  • Violations in error prevention & recovery
  • No smart defaults or saved data
  • Bad support for recognition vs recall

Ideation

🧠 UX Opportunities

  • Simplify refund lookup process
  • Preview before payment
  • Smart help search (voice, FAQ)

📱 Prototypes

  • Pre-filled hints, masked SSN
  • Processor comparison + modal
  • Help flow with search + voice

Solution Highlights

✨ Final Touches

  • Guest login mode
  • WCAG-compliant UI
  • Onboarding to guide first use

Usability Testing

👥 Participants

11 users, ages 16–65, mixed experience levels

🧪 Methods

  • Think-Aloud Protocol
  • System Usability Scale (SUS)

📊 Results

  • Guest login preferred
  • Help simplified to search + voice
  • Refund flow success rate ↑

✅ Final Outcome

Users completed tasks faster, with less hesitation, and gave higher usability scores.
